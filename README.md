# 일정 취합 사이트

여러 사람의 가능한 시간을 30분 단위 표로 모아 한눈에 보는 도구입니다.
Next.js(React) + API 라우트로 만들어졌고, 데이터는 `lib/db.js` 한 곳에서 관리합니다.

## 기능

- **프로젝트 추가**: 목록 화면 오른쪽 위 `+` 버튼
- **사람 추가**: 프로젝트 안에서 `+ 사람 추가` 탭
- **이름·일정 수정/저장**: 프로젝트 이름, 사람 이름은 클릭 후 수정하면 자동 저장
- **사람별 탭**: 탭을 누르면 그 사람의 일정을 색칠된 표로 확인·편집
- **표 설정**: 받을 날짜 범위(시작일~종료일)와 시간 범위(시작~종료)를 정하면 30분 단위 표가 만들어짐
- **클릭/드래그 선택**: 칸을 클릭하거나 드래그해 칠하면 선택, 다시 칠하면 취소
- **일별 메모**: 날짜 바로 아래 칸에 그 날의 메모 입력
- **전체 취합 뷰**: 모든 사람의 가능 시간을 겹쳐서 인원수를 색 농도로 표시 (마우스를 올리면 가능한 사람 이름 표시)
- **공유 링크**: 프로젝트 주소를 복사해 다른 사람에게 전달하면 각자 자기 탭에 입력 가능
- 다른 사람이 입력하면 약 4초마다 자동으로 화면에 반영됩니다.

## 로컬에서 실행

```bash
npm install     # 처음 한 번
npm run dev      # 개발 서버 (http://localhost:3000)
```

운영용으로 실행하려면:

```bash
npm run build
npm run start
```

데이터는 프로젝트 폴더의 `data.json` 파일에 저장됩니다.

## 인터넷에 배포

저장소가 **환경변수에 따라 자동으로 바뀝니다.**

| 환경변수 `DATABASE_URL` | 저장소 |
| --- | --- |
| 없음 | `data.json` 파일 (로컬 개발 / 영구 디스크 호스팅) |
| 있음 | Postgres (Vercel + Neon 등 서버리스 배포) |

**Vercel + Neon으로 배포하는 자세한 순서는 [DEPLOY.md](DEPLOY.md) 참고.**
요약: Neon에서 무료 DB 만들기 → 연결 문자열 복사 → Vercel에 배포하고
`DATABASE_URL` 환경변수에 그 문자열 등록. 코드 수정은 필요 없습니다.

> Render / Railway 등 **영구 디스크가 있는 서버**라면 `DATABASE_URL` 없이
> `npm run build && npm run start`로 그대로 띄워도 됩니다. (`DATA_FILE`로 저장 위치 지정 가능)

## 폴더 구조

```
app/
  page.js                 프로젝트 목록 화면
  p/[id]/page.js          프로젝트 화면 (탭·설정·공유)
  api/...                 REST API 라우트
components/
  ScheduleGrid.js         30분 단위 표 (클릭/드래그 선택 + 메모)
  OverviewGrid.js         전체 취합 히트맵
lib/
  db.js                   데이터 저장 계층 (여기만 바꾸면 DB 교체 가능)
  schedule.js             날짜/시간 격자 생성
  api.js, ui.js           클라이언트 헬퍼
```
